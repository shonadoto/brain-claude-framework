# /digest — выжимка контекста из завершённых сессий

Корень brain (далее `$BRAIN`) — содержимое файла `~/.claude/brain-dir`; файла нет → `~/brain`.

Страховочный контур автосохранения: разбирает транскрипты сессий, залогированных SessionEnd-хуком, и дописывает упущенное в журналы задач. Правила записи — `$BRAIN/CONVENTIONS.md` (прочитай раздел «Правила по слоям» перед началом).

## Файлы

- `$BRAIN/inbox/sessions.log` — TSV, пишется хуком: `дата \t session_id \t cwd \t transcript_path \t размер \t reason`. Append-only, НЕ редактировать.
- `$BRAIN/inbox/sessions-digested.log` — session_id обработанных сессий, по одному на строку. Ведёт этот скилл.

## Шаги

### 1. Собрать необработанные сессии

Session_id из `sessions.log`, отсутствующие в `sessions-digested.log`. Для каждой проверить:

- транскрипт существует (мог быть удалён ротацией) — нет → пометить digested с пометкой `missing`, в отчёт;
- размер < 20000 байт → скорее всего тривиальная сессия: пометить digested с пометкой `trivial`, в отчёт одной строкой (НЕ читать транскрипт);
- сессии текущего разговора не трогать.

Необработанных нет → «нечего дайджестить», конец.

### 2. Извлечь выжимку из каждого транскрипта

На каждый транскрипт — отдельный сабагент (транскрипты большие, не читать в основной контекст). В промпт сабагенту передать **точный путь** транскрипта и попросить вернуть структурированно:

- **задача**: из cwd, упоминаний в диалоге или имён файлов brain; не определяется → `unknown`
- **решения**: что выбрано и почему, что отвергнуто
- **тупики**: «пробовали X — не работает из-за Y»
- **внешние факты**: сказал пользователь/коллега, выяснилось из документа
- **этапы**: документ готов, ответ отправлен, договорённость достигнута
- **кандидаты в паттерны**: повторяющиеся ручные действия, коррекции пользователя, явные одобрения приёма («вот так правильно» — закрепляемый позитивный паттерн), диагностированные грабли инструментов/окружения класса «повторится» — даже без коррекции пользователя
- **открытые петли**: незакрытые обещания/хвосты («не забыть X», «жду ответа от Y», вопрос без ответа), оставшиеся открытыми к концу сессии

Сабагенту: только чтение, ничего не писать; вернуть пусто, если сессия не содержала рабочего контекста (болталка, разовый вопрос).

Конец транскрипта НЕ интерпретировать как «сессия оборвалась/прервана»: sessions.log логирует и паузы — сессия может продолжаться позже. Статус сессии из транскрипта не выводим; в выжимках и записях формулировать «вопрос остался открытым / работа не начата», не «сессия оборвалась на X».

### 3. Свести с журналами (дедуп)

Для каждой задачи прочитать существующий `tasks/<task>/journal.md` (нет в `tasks/` → проверить `tasks/archive/<task>/` — закрытые живут там, дописывать в архив, не создавать дубль): записи, которые агент уже сделал по ходу сессии, НЕ дублировать. Дайджест добавляет только упущенное.

- Задача `unknown`, но контекст ценный → спросить пользователя, куда отнести (пачкой, в конце).
- Кандидаты в паттерны → `learning/patterns.md` (наблюдение со ссылкой на session_id и дату). Коррекции пользователя без записанного правила → отдельная секция отчёта «незафиксированные уроки».
- Открытые петли: задачные → проверить, что они отражены в «Что осталось» CONTEXT.md задачи; незадачные → предложить в `loops.md` (сверить с уже записанными — не дублировать). Закрытые по факту транскрипта петли из `loops.md` → предложить отметить `[x]`.

### 4. Показать перед записью

**Обязательно**: показать пользователю предлагаемые дописки (по задачам) и дождаться подтверждения через AskUserQuestion (записать всё / выбрать / отменить). Не писать молча. Если дописок много (несколько задач / новые правила) — сначала полный текст отдельным ходом, вопрос следующим ([[rules/overview-before-question]]).

### 5. Записать

После подтверждения:

1. Дописать в `tasks/<task>/journal.md` (создать директорию + строку в `INDEX.md`, если задачи ещё нет). Источник записи: `digest, сессия <session_id>, <дата>`.
2. CONTEXT.md задачи: **нет** (journal есть, дистиллята нет) → предложить **создать** дистиллят из журнала в этом же прогоне; **есть, но заметно разошёлся** с журналом (новые решения/статус/хвосты) → предложить **обновить**. Дистиллят перегенерируемый, ~экран (цель, статус, ключевые решения, открытые вопросы, грабли), строго по журналу — ничего, чего в нём нет ([[rules/no-fabrication]]: CONTEXT.md — view журнала, не источник).
2a. У задачи в frontmatter CONTEXT.md есть `project: <id>` и сессия содержала этапное событие? → освежить строку этой задачи в `projects/<id>/CONTEXT.md` (таблица задач + дата снапшота) и хвосты проекта, если изменились. Решения/детали в проект не копировать — они остаются в journal задачи (анти-дубль, CONVENTIONS). Создание/обновление — под тем же подтверждением, что и дописки журнала (п. 4).
3. Дописать обработанные session_id в `sessions-digested.log`.
4. Закоммитить и запушить: `cd $BRAIN && git add -A && git commit -m "digest: <дата>" && git push` (пуш — если настроен remote; [[rules/brain-backup]]).

### 6. Отчёт

- таблица: сессия → задача → что дописано (или trivial/missing/пусто)
- незафиксированные уроки (коррекции без правила) — отдельно
- кандидаты в паттерны — отдельно

## Запреты

- Не редактировать `sessions.log` и прошлые записи журналов (append-only).
- Не читать транскрипты целиком в основной контекст — только через сабагентов.
- Не писать в журналы без показа пользователю (п. 4).
- Не создавать правила/скиллы из кандидатов в паттерны — только наблюдения в `patterns.md`; решения принимает `/retro` + пользователь.
