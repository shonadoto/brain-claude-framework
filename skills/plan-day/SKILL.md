# /plan-day — план дня

Корень brain (далее `$BRAIN`) — содержимое файла `~/.claude/brain-dir`; файла нет → `~/brain`.

Собирает открытое из brain и превращает в реалистичный план на сегодня. План — предложение, а не приговор: пользователь правит, агент фиксирует.

## Шаги

1. **Собрать сырьё** (только чтение): открытые петли из `loops.md` (секции Сделать/Жду/Вопросы), «Что осталось» активных задач (`tasks/*/CONTEXT.md`), хвосты зон (`areas/*/CONTEXT.md`, если есть), вчерашний план `days/<вчера>.md` (перенесённое и несделанное), количество неразобранных заметок inbox.
2. **Спросить пользователя** (одним AskUserQuestion): сколько сегодня времени на работу с хвостами (мало/полдня/весь день) и сколько энергии (низкая/нормальная/высокая). Пользователь уже сказал в запросе — не переспрашивать.
3. **Составить план** по корзинам, реалистично под ответы:
   - **Главное (1–3)** — двигает активные задачи; при низкой энергии — максимум одно.
   - **Стоит сделать** — петли «Сделать», протухающие «Жду» (пнуть), вопросы.
   - **Быстрые победы** — мелочь ≤15 минут (включая «разобрать inbox», если накопилось).
   Каждый пункт — со ссылкой на источник (задача/петля), человекочитаемо ([[rules/ids-human-context]]).
4. **Показать план** пользователю; после правок записать в `$BRAIN/days/<YYYY-MM-DD>.md` (создать `days/`, если нет):

   ```markdown
   # День YYYY-MM-DD
   План (энергия: X, время: Y):
   ## Главное
   - [ ] ...
   ## Стоит сделать
   - [ ] ...
   ## Быстрые победы
   - [ ] ...
   ```
5. Закоммитить и запушить, если git настроен ([[rules/brain-backup]]).

## Запреты

- Не выдумывать пункты — только из реальных записей brain ([[rules/no-fabrication]]). Пользователь назвал новое дело → добавить и в план, и (если это петля/задача) в нужное место brain.
- Не перегружать: план должен помещаться в названное время; остальное — явно «не сегодня».
