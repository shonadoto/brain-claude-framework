# /update-brain — обновление фреймворка

Корень brain (далее `$BRAIN`) — содержимое файла `~/.claude/brain-dir`; файла нет → `~/brain`.

Подтягивает свежую версию механики фреймворка (скиллы, хуки, скрипты, шаблоны) из репозитория в существующий brain. **Данные пользователя (журналы, знания, петли, правила, профиль) не трогаются.**

## Шаги

1. Показать текущую и доступную версии:
   ```bash
   cat $BRAIN/.claude/VERSION 2>/dev/null; cat $BRAIN/.claude/.update-cache 2>/dev/null
   ```
2. Склонировать свежий репозиторий во временную директорию и запустить установщик обновления:
   ```bash
   tmp=$(mktemp -d) && git clone --depth 1 https://github.com/shonadoto/brain-claude-framework "$tmp" && bash "$tmp/update.sh"
   ```
3. Показать пользователю вывод `update.sh` своими словами: что обновлено, какие новые скиллы/хуки появились, какие файлы репозитория разошлись с локальными (CONVENTIONS.md, стартовые правила) — их update.sh не перезаписывает, только сообщает.
4. Убрать временную директорию: `rm -rf "$tmp"`.
5. Если git настроен — изменения закоммитить и запушить: `cd $BRAIN && git add -A && git commit -m "update: фреймворк <версия>" && git push` ([[rules/brain-backup]]).
6. Напомнить: новые скиллы и хуки подхватываются при перезапуске Claude Code.

## Правила

- Расхождения данных-файлов (CONVENTIONS.md, существующие правила) НЕ перезаписывать — показать пользователю diff-команду из вывода update.sh, решает он.
- Сети нет / clone упал → сказать честно и остановиться, не изобретать обходных путей.
