# /brain-doctor — диагностика установки

Корень brain (далее `$BRAIN`) — содержимое файла `~/.claude/brain-dir`; файла нет → `~/brain`.

## Шаги

1. Запустить диагностику:
   ```bash
   bash $BRAIN/.claude/scripts/doctor.sh
   ```
   Скрипта нет (старая установка) → диагностируй вручную по тому же списку: brain-dir, структура директорий, симлинк `~/.claude/rules`, симлинки скиллов, хуки в `~/.claude/settings.json`, jq/git/curl, git remote, остатки незаменённого плейсхолдера установщика.
2. Пересказать результат пользователю простыми словами: что работает, что сломано и **чем это грозит** (например: «хук session-end не зарегистрирован → сессии не логируются → /digest не увидит прошлые разговоры»).
3. Для каждой проблемы — предложить конкретное лечение (команды из вывода doctor.sh). Чинить — только с согласия пользователя; набор правок показать заранее.

## Правила

- Сам скрипт ничего не меняет — это диагностика. Любая починка — отдельные команды с подтверждением.
- Не выдумывать состояние — только то, что показал скрипт/проверки ([[rules/no-fabrication]]).
