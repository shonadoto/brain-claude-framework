# /close-day — закрытие дня

Корень brain (далее `$BRAIN`) — содержимое файла `~/.claude/brain-dir`; файла нет → `~/brain`.

Парный к `/plan-day` вечерний ритуал: свести день, ничего не потеряв.

## Шаги

1. Прочитать `$BRAIN/days/<сегодня>.md`. Плана нет → спросить пользователя, чем занимался день, и сразу к п. 3.
2. Пройтись по пунктам плана с пользователем (компактно, не по одному: показать список, спросить «что сделано? что переносим?»).
3. Дописать в файл дня секцию:

   ```markdown
   ## Итог (записано HH:MM)
   - сделано: ...
   - не сделано → перенесено: ... (куда)
   - заметки дня: ...
   ```
4. **Разнести последствия** (с показом пользователю, без молчаливых правок):
   - сделанные петли → `[x]` с датой в `loops.md`;
   - сделанное по задачам → строка в `tasks/<task>/journal.md`, обновить «Что осталось» в CONTEXT.md;
   - несделанное «Главное» → остаётся в хвостах задачи/петлях (не дублировать в days);
   - новые хвосты, всплывшие за день, → `loops.md` или CONTEXT задачи.
5. Одной строкой — заготовка на завтра: «главный кандидат на завтра: X» (в конец файла дня).
6. Закоммитить и запушить, если git настроен ([[rules/brain-backup]]).

## Запреты

- Не закрывать петли и пункты без слов пользователя.
- Файлы прошлых дней не редактировать — только сегодняшний (история дней append-only по дням).
