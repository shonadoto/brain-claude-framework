# /sync-brain — бэкап brain в git

Корень brain (далее `$BRAIN`) — содержимое файла `~/.claude/brain-dir`; файла нет → `~/brain`.

Догоняющий бэкап: в норме каждое изменение brain коммитится и пушится сразу ([[rules/brain-backup]]). Эта команда — для ручного догона (незакоммиченные хвосты, упавший пуш).

## Шаги

1. Проверка, что brain — git-репозиторий:
   ```bash
   cd $BRAIN && git rev-parse --is-inside-work-tree
   ```
   Нет → сказать пользователю: «brain не под git; настроить бэкап — см. README фреймворка, раздел "Бэкап"» и закончить.
2. Статус:
   ```bash
   cd $BRAIN && git status --short
   ```
3. Есть незакоммиченное → закоммитить одним коммитом:
   ```bash
   cd $BRAIN && git add -A && git commit -m "sync: <дата> — <короткая сводка что изменилось>"
   ```
   (сводку построй по `status`: «журналы X, Y; правило Z»)
4. Пуш (если настроен remote):
   ```bash
   cd $BRAIN && git push
   ```
   Remote не настроен → сказать одной строкой, что коммиты локальные, и как добавить приватный remote (см. README, раздел «Бэкап»).
5. Ответ одной строкой: сколько коммитов ушло, куда.

## Правила

- Перед коммитом глазами пробеги diff на секреты (токены, пароли, чужие ПД) — секретов в brain быть не должно (CONVENTIONS): нашёл → стоп, показать пользователю, не пушить.
- Конфликт/расхождение при пуше → `git pull --rebase`, расхождения показать пользователю, не резолвить молча.
- Remote должен быть **приватным** — brain содержит личный рабочий контекст.
