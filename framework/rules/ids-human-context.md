# Ссылки и ID — всегда с человекочитаемым описанием

Когда упоминаю в разговоре с пользователем документ, задачу, сессию или любой ID — **никогда не давать голый номер/имя**. Рядом всегда: (1) короткое описание сути и (2) кликабельная ссылка, если она есть: «задача `otchet-q2` (квартальный отчёт для правления)», а не просто «otchet-q2».

## Why

Идентификатор для агента — ключ, по которому он держит контекст; для пользователя голый ID ничего не значит — он не помнит наизусть, что за ним стоит, и вынужден идти искать, чтобы понять фразу. Это трение и потеря нити разговора.

## How to apply

- Первое упоминание сущности в сообщении/блоке: `<id> (короткая суть)`. Если упоминаний много вперемешку — держать описание при каждом.
- В таблицах/отчётах (`/status`, `/digest`) — колонка или скобка с человеческим описанием рядом с ID.
- Источник описания — `INDEX.md` / `CONTEXT.md` / journal: там у каждой сущности есть строка сути. Брать оттуда, не выдумывать ([[no-fabrication]]).
- Исключение: пользователь сам только что назвал сущность и явно держит контекст — тогда можно короче. По умолчанию — с описанием.

Связано: [[language]] (читаемость для пользователя), [[no-fabrication]].
