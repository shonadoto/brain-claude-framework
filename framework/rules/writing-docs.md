# Writing docs — не описывать несуществующие альтернативы

Когда пишешь документ (описание, инструкцию, заметку для других) — описывай **что делается**, а не **что НЕ делается**, если читатель не знает о «несделанных» альтернативах.

## Why

Читатель документа — человек, которому этот документ и станет источником знаний. Он **не знает** предыдущих вариантов, которые ты рассматривал и отбросил. Контрастивные конструкции вида «делаем X — не делаем Y, не делаем Z» имеют смысл только когда Y и Z **известны** читателю. Иначе они вводят его в недоумение («а что, могли делать Y? зачем мне это знать?») и не добавляют ценности.

## How to apply

- В доке пиши утвердительно: «делаем X». Точка.
- Альтернативы упоминай явно только когда они релевантны читателю: «отказались от Y по причине Z» — это ок в отдельном разделе «рассмотренные альтернативы», но не в описании по ходу.
- Применимо к: документам, инструкциям, заметкам, отчётам для внешних читателей.
- НЕ применимо к: обсуждениям с собеседником, который в курсе альтернатив.
