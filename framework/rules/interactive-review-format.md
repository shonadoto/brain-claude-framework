# Интерактивное ревью списка — AskUserQuestion со счётчиком, не текстовая простыня

Когда пользователь просит пройтись по списку из N элементов с его решением по каждому («пройдись по списку, я выберу», «опроси меня по каждому», «давай по одному») — вести через AskUserQuestion по элементу (или небольшой пачке) со счётчиком прогресса (`3/16`), а не вываливать весь список текстом с просьбой ответить.

## Why

Текстовый дамп N элементов заставляет пользователя самому вести учёт («по первому да, по третьему нет, до куда мы дошли?») — это и есть работа, которую он делегировал. AskUserQuestion с вариантами и счётчиком держит фокус на одном решении за раз и показывает прогресс.

## How to apply

- Серия однотипных мелких решений (оставить/удалить, да/нет, выбор категории) по списку → AskUserQuestion по одному элементу или пачке 2-4, в заголовке/тексте — счётчик `i/N`.
- В вопросе давать достаточный контекст для решения по элементу (что это, зачем, последствие выбора) — чтобы не приходилось спрашивать «а что это такое?».
- НЕ противоречит [[overview-before-question]]: там — одно решение, требующее прочтения большого контента (сначала контент, потом вопрос); здесь — серия мелких решений, где контент каждого помещается в сам вопрос.
- Пользователь явно попросил «просто покажи список» — показать списком, не дробить.

Связано: [[overview-before-question]].
