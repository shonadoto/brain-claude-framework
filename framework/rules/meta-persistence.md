# Persistence rule — куда сохранять «запомни»

Когда пользователь говорит **«запомни»** или просит запомнить durable-факт (предпочтения, договорённости, правила, имена, как устроено X), сохраняй в brain — не в эфемерную память сессии.

Это надёжно: `~/.claude/rules/` — симлинк на `__BRAIN_DIR__/rules/`, поэтому правила автозагружаются в начале каждой сессии; brain коммитится в git и пушится в приватный remote ([[brain-backup]]) — сам файл и есть durable backup.

## Куда именно

| Что | Куда |
|---|---|
| Правило поведения Claude (сквозное) | `rules/<topic>.md` |
| Правило с узким триггером («когда делаю X») | `rules-ondemand/<topic>.md` + строка в `rules/ondemand-rules.md` |
| Подтверждённый факт «как устроено X» | `knowledge/<topic>.md` (frontmatter `status: verified`, источник) |
| Личное предпочтение / профиль | секция «Профиль пользователя» в `.claude/CLAUDE.md` |
| Хвост / «не забыть» | `loops.md` |
| Контекст конкретной задачи | `tasks/<task>/journal.md` |

Если подходящего файла нет — создай новый с описательным kebab-case-именем, выбрав слой:
- **`rules/`** (автозагрузка в каждую сессию) — сквозные поведенческие правила, нарушение которых ловится только в моменте — то, что агент должен знать ДО входа в опасную зону;
- **`rules-ondemand/`** (чтение при входе в тему) — правила с явным триггером. Обязательно: строка с триггером и ядром в таблице `rules/ondemand-rules.md`, иначе правило не найдут.

После записи — показать пользователю путь и формулировку (не просто «запомнил»).
