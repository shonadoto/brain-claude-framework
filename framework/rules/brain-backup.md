# Brain — коммитить изменения сразу, пушить без вопросов

Если brain — git-репозиторий: после содержательных изменений (журналы, правила, знания, петли) — сразу `git add -A && git commit`, и если настроен remote — `git push`, не спрашивая пользователя. Локальных непушнутых коммитов накапливаться не должно.

## Why

brain — единственная копия рабочего контекста; незакоммиченные изменения — риск потери, а вопрос «закоммитить?» в каждой сессии — лишнее трение. Пуш в приватный remote — мгновенный durable-бэкап.

## How to apply

- После прогона скиллов, пишущих в brain (`/digest`, `/retro`, `/inbox-process`, `/evolve`), и после ручных правок правил/знаний: `cd __BRAIN_DIR__ && git add -A && git commit -m "<скилл/тема>: <дата>" && git push`.
- Remote не настроен или brain не git-репозиторий → пропустить молча (не предлагать настроить в каждой сессии).
- Перед пушем пробежать diff на секреты (токены, пароли, чужие ПД) — нашёл → стоп, показать пользователю, не пушить (CONVENTIONS, «Никаких секретов»).
- Push упал (сеть, конфликт) → `git pull --rebase`, конфликты не резолвить молча — показать пользователю. Догнать можно командой `/sync-brain`.
- Серия правок в одном прогоне — допустимо закоммитить и запушить один раз в конце прогона.
