# /evolve — мета-контур: улучшение самого процесса обучения

Разница с `/retro`: retro учится ИЗ работы (паттерны, коррекции, violations), evolve улучшает МЕХАНИЗМЫ обучения — слои памяти, скиллы, хуки, конвенции — на основе внутренних болей и внешних источников. Перезапускаемый: каждый прогон инкрементален (память прогонов, отклонённое не предлагается заново).

## Файлы

- `learning/evolve-runs.log` — журнал прогонов: `дата \t период \t направления ресёрча \t итог`. Append-only, ведёт этот скилл.
- `learning/evolve-considered.md` — все рассмотренные идеи со статусами: внедрено / отклонено (причина) / отложено (условие возврата). Анти-повтор.
- `learning/proposals.md` — трекеры внедрения.
- Объекты улучшения: `CONVENTIONS.md`, `rules/`, `rules-ondemand/`, `.claude/skills/`, `.claude/hooks/`, `~/.claude/settings.json` (хуки).

## Шаги

### 1. Память прогонов

Прочитать `evolve-runs.log` (период = с прошлого прогона; лога нет → первый прогон), `evolve-considered.md` целиком, хвост `proposals.md` (незакрытые пункты прошлых трекеров — кандидаты на доделку в первую очередь).

### 2. Внутренний аудит — где болит сейчас (Explore-сабагент)

Сабагенту — конкретные команды, не «осмотрись»:

- бюджет автозагрузки: `du -sb __BRAIN_DIR__/rules/` (ориентир ≤150 КБ) + топ-5 файлов по размеру;
- правила с новыми `violations:` за период (grep по frontmatter) — текстовая форма не работает, кандидаты на хук/перенос;
- `learning/patterns.md` — паттерны без закрывающего механизма; `learning/proposals.md` — застрявшие «ждёт решения»;
- очередь digest (`sessions.log` − `sessions-digested.log`), CONTEXT.md старше 7 дней, давность последнего `/retro` (`retro-runs.log`);
- хуки `.claude/hooks/` + `jq '.hooks' ~/.claude/settings.json` — все ли зарегистрированы, нет ли мёртвых;
- целостность ссылок: запустить `__BRAIN_DIR__/.claude/scripts/lint-brain.sh`.

Выход: список болей с цифрами. Боль = измеренный факт, не мнение ([[rules/no-fabrication]]). Факты «файла нет» от сабагента перепроверять лично.

### 3. Внешний ресёрч — параллельные сабагенты

⚠ **Privacy-гард**: в веб-запросы — только обезличенные темы («file-based agent memory», «rule lifecycle», «context engineering»). НИКОГДА не вставлять тексты своих правил, имена людей, компаний, рабочие данные.

a) **Свежие работы и практики** (general-purpose + WebSearch/WebFetch): публикации по памяти агентов / continual learning / context engineering за период; инженерные блоги. На каждую идею: механизм в 2-3 предложениях + применимость к file-based markdown-памяти + ссылка-источник.

b) **Новые фичи харнеса** (claude-code-guide): whats-new/changelog Claude Code за период; новое в memory / hooks / skills / subagents / context management — что из этого brain недоиспользует.

c) **Практики инструментов** (только если прошлый прогон >1 месяца назад): изменения в подходах к памяти у популярных ассистентов — дизайн-решения, не маркетинг.

### 4. Синтез предложений

- Идея уже в `evolve-considered.md` со статусом «отклонено» и новых аргументов нет → пропустить (упомянуть строкой в отчёте).
- Идея не отвечает ни одной боли из шага 2 → в considered со статусом «отложено: нет текущей боли», в предложения не тащить. Механизмы ради механизмов не внедряем.
- Остальное → таблица: предложение / какую боль закрывает / цена / источник. Приоритет = ценность/цена.

### 5. Показать и спросить — разными ходами

Полный отчёт (боли → находки ресёрча → таблица предложений) и СТОП ([[rules/overview-before-question]]). Следующим ходом — AskUserQuestion (multiSelect: какие пункты внедрять).

### 6. Внедрить одобренное в той же сессии

- Только локальными дельтами — монолитная перезапись корпуса запрещена (CONVENTIONS п.10).
- Новые/изменённые хуки: pipe-тест КАЖДОЙ ветки до регистрации + живое доказательство, где возможно.
- После структурных изменений (scope правил, конвенции, settings.json) — верификация: grep битых ссылок, `jq -e` на settings.json, запуск изменённых скриптов.
- Перенос правила в `rules-ondemand/` → строка с триггером и ядром в `rules/ondemand-rules.md` обязательна.

### 7. Финализация

1. `evolve-considered.md` — дописать ВСЕ рассмотренные идеи этого прогона со статусами и источниками.
2. `proposals.md` — новый блок внедрения с чеклистом (или дописать незакрытый).
3. `evolve-runs.log` — строка прогона.
4. Коммит + пуш: `cd __BRAIN_DIR__ && git add -A && git commit -m "evolve: <дата> — <кратко>" && git push` ([[rules/brain-backup]]).
5. Отчёт: что внедрено / отклонено / отложено, новый размер бюджета автозагрузки.

## Запреты

- НЕ менять правила/скиллы/хуки/конвенции без явного одобрения из шага 5. Это касается и САМОГО этого скилла: предложения по изменению `/evolve` проходят тот же гейт.
- НЕ вставлять личные/рабочие данные в веб-запросы (шаг 3 — обезличенные темы).
- НЕ переписывать файлы целиком «для порядка» — только дельты с диффом.
- НЕ предлагать отклонённое повторно без новых аргументов (сверка с `evolve-considered.md`).
- Идея без ссылки-источника не предлагается; боль без измерения не заявляется ([[rules/no-fabrication]]).
- Регистрировать хук без pipe-тестов всех веток — запрещено.
