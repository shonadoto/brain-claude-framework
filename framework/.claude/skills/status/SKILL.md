# /status — сводка незавершённого

Один экран: всё, что открыто и может быть забыто. Только чтение и показ; правки `loops.md` — по явной реакции пользователя.

## Шаги

### 1. Собрать (всё — чтение, без сабагентов)

1. **Петли**: `__BRAIN_DIR__/loops.md` — открытые `[ ]` по секциям. Для «Жду» посчитать возраст от даты записи (сегодня — `date '+%Y-%m-%d'`); > 5 дней → пометить ⚠ протухло.
2. **Задачи**: секция «Задачи» в `INDEX.md` → для каждой прочитать `tasks/<task>/CONTEXT.md`: взять Этап + «Что осталось» + открытые вопросы. Секцию «Архив» INDEX и `tasks/archive/` НЕ обходить — закрытые живут там (CONVENTIONS, «Закрытие → архив»). `status: archived` в активной секции — рассинхрон: пропустить и пометить «переместить в архив». CONTEXT.md нет → показать задачу с пометкой «нет CONTEXT.md» (хвосты неизвестны).
3. **Проекты**: секция «Проекты» в `INDEX.md` → для каждого прочитать `projects/<id>/CONTEXT.md`: открытые хвосты одной строкой + дата снапшота таблицы задач. Дата старше 7 дней → пометить «⚠ карта протухла (дата)».
4. **Inbox**: файлы `inbox/*.md` вне `archive/` (не считая `sessions*.log`) → сколько необработанных заметок.
5. **Сессии**: строки `sessions.log` минус `sessions-digested.log` → сколько ждёт `/digest`.
6. **Proposals**: `learning/proposals.md` → пункты со статусом «ждёт решения».

### 2. Показать

Компактно, в этом порядке (важное сверху):

```
## Петли
Сделать (N): ...
Жду (N): ... [⚠ 7 дней]
Вопросы (N): ...
Потом (N): ...

## Задачи
<task> — этап | что осталось (1 строка)

## Проекты
<id> — хвосты (1 строка) | снапшот YYYY-MM-DD [⚠ протух]

## Гигиена brain
inbox: N заметок не разобрано (→ /inbox-process)
сессии: N ждут /digest
proposals: N ждут решения
```

Пустые секции — одной строкой «пусто», не раздувать. Закрытые петли не показывать. У каждой задачи/петли — человекочитаемое описание, не голый ID ([[rules/ids-human-context]]).

### 3. Реакция пользователя

- «сделал X», «это закрыто» → `[x]` + дата закрытия в `loops.md`.
- «добавь Y» → строка в нужную секцию (дата + суть + источник).
- Закрытых `[x]` стало > ~15 → предложить перенести их в `loops-archive.md` (не делать молча).
- Правки `loops.md` закоммитить и запушить: `cd __BRAIN_DIR__ && git add loops.md loops-archive.md 2>/dev/null; git commit -m "loops: <дата>" && git push` (если есть изменения и настроен git; [[rules/brain-backup]]).

## Запреты

- Не выдумывать хвосты — только то, что реально записано в файлах.
- Не закрывать и не удалять петли без слов пользователя.
- Не дублировать задачные хвосты в loops.md — они живут в CONTEXT.md задач.
