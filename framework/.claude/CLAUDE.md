# Brain — единая точка входа Claude Code

Все взаимодействие с Claude живёт в `__BRAIN_DIR__`: правила (`rules/`, автозагрузка через симлинк), знания (`knowledge/`), скиллы (`.claude/skills/`), контексты задач. Правила работы с хранилищем — `__BRAIN_DIR__/CONVENTIONS.md`; каждый агент, пишущий в brain, обязан их соблюдать.

## Журналирование контекста задач

При работе над задачей дописывай в `__BRAIN_DIR__/tasks/<task>/journal.md` ПО ХОДУ сессии — немедленно при событии:

1. принято решение (выбран подход, отвергнута альтернатива) — решение + почему
2. найден тупик («X не работает из-за Y»)
3. получен внешний факт (коллега сказал, пользователь уточнил, выяснилось из документа)
4. завершён значимый этап (план готов, документ отправлен, ответ получен)

Запись — 1–3 строки с датой. Рутинные действия не журналируются. Формат и остальные правила — `__BRAIN_DIR__/CONVENTIONS.md`. Директории задачи нет — создай (`journal.md` + строка в `INDEX.md`).

Задача входит в проект (frontmatter `project:` в её CONTEXT.md)? При значимом этапе — в той же сессии обнови строку задачи в `projects/<id>/CONTEXT.md` (таблица задач + дата снапшота). Инфу не дублировать: решения одной задачи — в journal задачи, межзадачные — в проекте (CONVENTIONS, «Граница task / project»).

При старте работы над знакомой задачей — сначала прочитай `tasks/<task>/CONTEXT.md` и хвост `journal.md`.

## Открытые петли (не забыть)

Пользователь говорит «не забыть X», «надо будет потом Y», «жду от Z ответа», «спросить у W» — сразу допиши строку (дата + суть + источник) в нужную секцию `__BRAIN_DIR__/loops.md` (Сделать / Жду / Вопросы / Потом). Хвосты по текущей задаче — в `tasks/<task>/CONTEXT.md`, не в loops. Сводка незавершённого — `/status`.

## Фиксация коррекций (контур обучения)

Коррекция пользователя («опять...», «я же говорил», tool-rejection с упрёком) → в том же turn: диагноз с подтверждением → правило в `__BRAIN_DIR__/rules/` (Why + How to apply) → показать путь и формулировку. Повторная ругань по существующему правилу → дописать дату в `violations:` правила. Подробно — раздел «Контур обучения» в CONVENTIONS.md.
