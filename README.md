# brain-claude-framework

Личная память и контур обучения для [Claude Code](https://claude.com/claude-code). Превращает Claude из «умного чата, который всё забывает» в ассистента, который помнит твои задачи, договорённости и предпочтения — и учится на своих ошибках.

Не требует навыков программирования: вся «память» — обычные текстовые файлы (markdown), которые можно читать и править хоть Блокнотом, хоть [Obsidian](https://obsidian.md).

## Идея

Claude Code сам по себе ничего не помнит между разговорами. Этот фреймворк даёт ему **brain** — папку с файлами, куда Claude по ходу работы записывает:

- **журналы задач** — что решили, почему, какие тупики прошли;
- **знания** — проверенные факты («как на самом деле устроено X»);
- **открытые петли** — «не забыть», «жду ответа от...», «спросить у...»;
- **правила** — уроки из твоих поправок: поправил Claude один раз — он записывает правило и больше так не делает.

В начале каждого нового разговора Claude автоматически подхватывает правила и видит сводку: сколько незавершённого, что протухло, что разобрать.

## Структура brain

```
brain/
├── inbox/            # сырые заметки — кидай всё подряд, разберётся потом
├── tasks/<задача>/   # журнал + краткий контекст по каждой задаче
├── projects/<id>/    # контекст проектов (несколько связанных задач)
├── knowledge/        # проверенные факты
├── learning/         # наблюдения и предложения по улучшению
├── loops.md          # открытые петли: сделать / жду / вопросы / потом
├── rules/            # правила Claude (автозагружаются в каждый разговор)
├── INDEX.md          # карта всего
└── CONVENTIONS.md    # правила ведения хранилища (для агентов)
```

«Задача» — любая именованная активность: отчёт, сделка, переговоры, исследование. Фреймворк не привязан к программированию.

## Команды

| Команда | Что делает |
|---|---|
| `/setup-brain` | первичная настройка: знакомство, профиль, первые задачи |
| `/inbox <текст>` | мгновенно записать заметку (ноль вопросов) |
| `/inbox-process` | разобрать накопленные заметки по местам |
| `/status` | один экран всего незавершённого |
| `/digest` | вытащить важное из прошлых разговоров в журналы задач |
| `/retro` | ретроспектива: чему Claude научился, какие уроки не зафиксированы |
| `/evolve` | улучшение самого фреймворка: аудит болей + поиск новых практик |
| `/sync-brain` | вручную забэкапить brain в git (обычно происходит само) |

## Установка

Потребуется:

- установленный [Claude Code](https://claude.com/claude-code);
- `jq` (утилита для работы с JSON: macOS — `brew install jq`, Ubuntu — `sudo apt install jq`);
- `git` — желательно, для бэкапа (без него тоже работает).

Дальше:

```bash
git clone https://github.com/shonadoto/brain-claude-framework.git
cd brain-claude-framework
./install.sh            # установит brain в ~/brain
```

Установщик скопирует заготовку, подключит правила/скиллы/хуки к Claude Code и подскажет следующие шаги. Свой путь: `./install.sh /путь/до/brain`; без git: `./install.sh --no-git`.

После установки **перезапусти Claude Code** и набери `/setup-brain` — Claude задаст несколько вопросов и настроит профиль под тебя.

## Ежедневный ритм

1. **Работаешь как обычно.** Claude сам пишет в журналы решения и договорённости по ходу дела.
2. **Пришла мысль** — `/inbox купить лицензию до пятницы`. Разберётся потом.
3. **Начало дня** — `/status`: что открыто, что протухло, что я кому должен.
4. **Раз в несколько дней** — `/inbox-process` и `/digest`: разобрать заметки, дожать упущенное из прошлых разговоров.
5. **Раз в неделю-две** — `/retro`: Claude показывает, где ты его поправлял, какие уроки записаны, что можно автоматизировать.

## Как Claude учится

Ключевой механизм — **контур обучения**. Когда ты поправляешь Claude («я же говорил...», «не делай так»), он обязан в тот же момент записать урок в `rules/` — с причиной и инструкцией, когда правило срабатывает. Правила автозагружаются в каждый новый разговор, поэтому урок переживает границу сессии.

`/retro` проверяет, что ни одна поправка не потерялась, и считает, какие правила работают, а какие нарушаются повторно (значит, сформулированы плохо — надо переформулировать или превратить в автоматическую проверку).

Фреймворк ставится с небольшим стартовым набором общих правил (`framework/rules/`): не выдумывать факты, подтверждать внешние действия, показывать контент до вопросов и т.п. Дальше набор растёт под тебя.

## Бэкап

brain — единственная копия твоей рабочей памяти. Настрой бэкап в **приватный** git-репозиторий:

1. Создай приватный репозиторий (например, на github.com — кнопка New repository, видимость **Private**).
2. Подключи его:

   ```bash
   cd ~/brain
   git remote add origin git@github.com:<твой-логин>/<имя-репо>.git
   git push -u origin main
   ```

После этого Claude бэкапит brain автоматически после каждого изменения.

**Важно:** brain содержит твой рабочий контекст. Репозиторий должен быть приватным; пароли и токены в brain не записываются вообще (это зашито в правилах хранилища).

## Удаление

1. Удали из `~/.claude/settings.json` блоки хуков с путями `<brain>/.claude/hooks/...`.
2. Удали симлинки: `~/.claude/rules` и `~/.claude/skills/{inbox,inbox-process,digest,status,retro,evolve,sync-brain,setup-brain}`.
3. Удали блок «Brain — точка входа» из `~/.claude/CLAUDE.md`.
4. Саму папку brain удаляй только если данные больше не нужны (или оставь как архив).

## Устройство репозитория

- `framework/` — заготовка brain, которая копируется при установке (внутри неё `__BRAIN_DIR__` — плейсхолдер, установщик заменяет его на реальный путь);
- `install.sh` — установщик.

Фреймворк после установки живёт своей жизнью: правь правила, конвенции и скиллы под себя — это твои файлы. Команда `/evolve` поможет улучшать их осознанно.
